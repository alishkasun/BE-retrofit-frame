Тест -дизайн Продукт.
Все поля обязательные.

GET-
1.Получение списка продуктов.

GET/id -
Позитивные  1. Получение данных по существующему id.
Негативные  2. Получение данных по не существующему id.
            3.Получение данных под неавторизованным пользователем.
            4.id = text

POST -
1.Отправка запроса с валидными данными.
    1.1 Валидные данные поля title
        1.1.1 На латинице
        1.1.2 На кирилице
        1.1.3 Символы
        1.1.5 Максимальное значение символов
        1.1.6 Минимальное значение символов

    1.2 Валидные данные поля price
         1.2.1 Числовые целые
         1.2.2 Числовые дробные
         1.2.3 Максимальное значение
         1.2.4 Минимальное значение

    1.3 Валидные данные поля categoryTitle
        1.3.1 На латинице
        1.3.2 На кирилице
        1.3.3 Символы
        1.3.4 Максимальное значение символов
        1.3.5 Минимальное значение символов

2.Отправка запроса с не валидными данными.
    2.1 Не валидные данные поля title
        2.1.1 Пустое значение
        2.1.2 null
        2.1.3 Пробел
        2.1.4 Числовые значения.

    2.2 Не валидные данные поля price
        2.2.1 Отрицательное значение
        2.2.2 С лидирующим 0
        2.2.3 Превышающее масимальное
        2.2.4 Ниже минимального
        2.2.5 Три знака после запятой.
        2.2.6 Ввод текста
        2.2.7 Пустое значение.
        2.2.8 null

    2.3 Не валидные данные поля categoryTitle
        2.3.1 Пустое значение
        2.3.2 null
         2.3.3 Пробел
         2.3.4 Числовые значения.

3.Создание под неавторизованным пользователем.

DELETE-
1.Удалени существующего id
2.Удалени не существующего id
3.Удалени ранее удаленного
4.Удалени под неавторизованным пользователем.
